Frequently Asked Questions

Notebookify is a website designed to give users top data about their Spotify listening habits and playlists. Notebooks can be customized and display short term, medium term, and long term data about your top tracks, artists, and genres. You can also generate graphs analyzing the contents of your individual playlists. Graphs include data such as how many times an artist appears, the tempo of tracks, how popular your playlist would be, and much much more!
No, Notebookify does NOT sell/use your Spotify data. We only ask you to log in so that we can use the Spotify API to get the top data/playlists that we can analyze. Your data is not stored anywhere and is safe.
If there is too much traffic on the site, Notebookify may not load properly. Also, if there is too much data to be analyzed in a playlist, the graphs may not load. Just try re-logging in with Spotify and hopefully it should work!
Notebookify uses the Spotify API to pull your top listening data as well as your playlists/tracks that can then be analyzed. Notebookify was created using Python (Flask) for the back-end and HTML, CSS (Bootstrap), and Javascript for the front-end. The Plotly Python library is used to generate the graphs.
The popularity score given to a track/artist is out of 100 (meaning that 0 is not well known at all while 100 is a very popular track/artist). These are provided by Spotify per track/artist in their database.
The graphs feature is a work in progress. Some bugs are still being worked out, but if there is a very very long playlist with a lot of data, there is a chance that the graphs page will not load. Sorry about that!
If there is too much data on a graph, it won't render properly on smaller screens (like mobile devices). The graphs feature is best viewed on a laptop/desktop with a larger screen.
